# ths_trade Agent 操作指南

## 设计目标

这套能力按 **搜索 / 决策 / 执行** 三层解耦：

1. **搜索层**：联网搜索公开信息、消息面、题材、板块热度
2. **决策层**：把用户自然语言解析成结构化交易意图，并做最基本校验
3. **执行层**：通过 `agent_trader.py`（市价委托）或 `limit_trader.py`（限价委托）把指令送入同花顺并自动确认到 `Y`

重点：
- **不写死股票、数量、策略编号**
- **支持买、卖、批量、查询**
- **支持自然语言下单入口**
- **市价委托和限价委托完全解耦**，走不同的脚本和服务端
- 默认走**极简执行模式**，避免额外查询导致乱点和无效操作

## 适用场景

当用户提出以下意图时，直接调用本 skill：
- 买入某只股票 / 卖出某只股票
- 模拟炒股 / 自动下单
- 批量买卖
- 查询当日委托或成交
- 先让我联网搜索，再按结论直接下单
- 用自然语言直接下达交易命令

## 前置条件

1. 同花顺 `xiadan.exe` 已打开并登录（模拟盘就选"模拟炒股"）
2. `D:\ths_trade\start.bat` 已启动交易服务
3. 同花顺窗口不要最小化

## Agent 调用方式

```python
import sys; sys.path.insert(0, r"D:\ths_trade")

# 市价委托（主板/创业板）
from agent_trader import buy, sell, batch, query, place_trade_text, preview_order, parse_trade_text

# 限价委托（科创板/北交所/主动限价）
from limit_trader import limit_buy, limit_sell
```

## 核心接口

### 1. 市价委托（agent_trader.py）

仅适用于**沪主板、深主板、创业板**，这些板块支持市价委托。

```python
# 市价委托买入
buy("300666", "江丰电子", 100)

# 市价委托卖出
sell("300666", "江丰电子", 100)
```

### 2. 限价委托（limit_trader.py）

适用于**科创板、北交所**（必须限价），以及**主板/创业板主动要求限价**的场景。

```python
# 科创板限价委托（最小200股）
limit_buy("688262", "国芯科技", 200, 35.78)

# 北交所限价委托
limit_buy("830001", "北交所股", 100, 10.00)

# 主板主动限价
limit_buy("601318", "中国平安", 100, 50.00)

# 限价卖出
limit_sell("688262", "国芯科技", 200, 40.00)
```

### 3. 批量下单（agent_trader.py）

```python
batch([
    {"code": "300666", "name": "江丰电子", "amount": 100, "operate": "buy"},
    {"code": "601318", "name": "中国平安", "amount": 100, "operate": "sell"},
])
```

### 4. 查询（agent_trader.py）

```python
query()
query("agent")
```

### 5. 自然语言解析（agent_trader.py）

```python
parse_trade_text("买江丰电子1手300666")
preview_order("sell", "601318", "中国平安", 100)
```

### 6. 自然语言直接执行（agent_trader.py）

```python
place_trade_text("卖出中国平安100股601318")
```

## 当前执行模式

当前默认使用**极简实盘/模拟盘执行模式**：
- 只做必要买卖操作
- 自动处理常见"委托确认"弹窗，并确认到 `Y`
- 不依赖额外查询结果才能完成下单
- 适合用户直接说"买这个 / 卖这个"时执行

## 推荐工作流

### 模式 A：用户直接交易

1. 用户说：`买江丰电子100股`
2. Agent 解析成结构化指令
3. 根据板块自动判断走市价/限价委托
4. 如信息完整，直接执行；如缺代码/数量/价格，先补问

### 模式 B：联网搜索后再交易

1. 先用联网工具搜索公开信息
2. 总结候选股票、板块逻辑、风险
3. 用户确认后，调用 `buy()` / `limit_buy()` 或 `place_trade_text()` 执行

### 模式 C：批量任务

1. 把用户意图拆成订单列表
2. 调用 `batch()` 一次提交
3. 如涉及不同标的，逐条确认代码和数量

## 返回格式

**buy/sell/batch/place_trade_text/limit_buy/limit_sell 返回**：
```json
{"code": 200, "msg": "success", "data": "agent"}
```
或
```json
{"code": 200, "msg": "success", "data": "agent_limit"}
```
表示指令已进入队列并开始执行自动化流程。

**query 返回**：
```json
{"code": 200, "msg": "success", "data": [...]} 
```

**错误返回**：
```json
{"code": -1, "msg": "ConnectionError ...", "data": null}
```
表示服务未启动或不可连接。

## 风险与校验

脚本内已做最基本校验：
- `operate` 只能是 `buy` / `sell`
- 股票代码必须是 6 位数字
- 股票名称不能为空
- **根据板块自动校验申报数量规则**（见下方板块规则表）
- **根据板块自动判断委托类型**（市价/限价，见下方规则）
- **限价委托必须提供价格，且价格>0**

建议交互时仍确认：
- 股票代码
- 股票名称
- 买卖方向
- 数量
- **委托类型**（市价/限价）
- **委托价格**（限价委托时）

## A股板块交易规则

脚本根据股票代码自动识别板块，并应用对应交易规则：

| 板块 | 代码前缀 | 最小申报 | 递增规则 | 市价委托 | 涨跌幅 |
|------|---------|---------|---------|---------|-------|
| 沪主板 | 60xxxx | 100股 | 100股整数倍 | 支持 | ±10% |
| 科创板 | 688xxx | **200股** | **超200股按1股递增** | **不支持** | ±20% |
| 深主板 | 00xxxx | 100股 | 100股整数倍 | 支持 | ±10% |
| 创业板 | 300xxx | 100股 | 100股整数倍 | 支持 | ±20% |
| 北交所 | 8xxxxx/4xxxxx | 100股 | 100股整数倍 | **不支持** | ±30% |

### 委托类型自动判断逻辑

1. **科创板股票（688开头）**：强制限价委托，必须提供 `price`，否则报错
2. **北交所股票（8/4开头）**：强制限价委托，必须提供 `price`，否则报错
3. **其他板块 + 指定了 price**：自动切换为限价委托
4. **其他板块 + 未指定 price**：默认市价委托

### 限价委托价格笼子规则

连续竞价阶段，限价委托价格需在有效申报范围内，超出范围将被交易所直接废单：

| 板块 | 买入上限 | 卖出下限 |
|------|---------|---------|
| 沪主板/深主板 | 基准价×102% 或 基准价+0.1元（取高） | 基准价×98% 或 基准价-0.1元（取低） |
| 创业板 | 基准价×102% 或 基准价+0.1元（取高） | 基准价×98% 或 基准价-0.1元（取低） |
| 科创板 | 基准价×102% | 基准价×98% |
| 北交所 | 基准价×105% 或 基准价+0.1元（取高） | 基准价×95% 或 基准价-0.1元（取低） |

注：
- 集合竞价阶段（9:15-9:25）仅受涨跌幅限制，不受价格笼子约束
- 新股上市前5个交易日无涨跌幅限制，但连续竞价仍受价格笼子约束
- 基准价 = 当前最优买/卖盘的即时价格

## 市价委托 vs 限价委托：架构对比

市价委托和限价委托**完全解耦**，走不同的脚本和服务端：

### 文件架构

```
D:\ths_trade\
├── agent_trader.py                          # 市价委托入口（命令行/API）
├── limit_trader.py                          # 限价委托入口（命令行/API）
├── applications\trade\
│   ├── Exec_Auto_Trade.py                   # 市价委托执行器
│   ├── Exec_Limit_Trade.py                  # 限价委托执行器
│   └── server\
│       ├── THS_Trader_Server.py             # 市价委托服务端（同花顺操作）
│       └── THS_Limit_Trader_Server.py       # 限价委托服务端（同花顺操作）
├── applications\Timer_Exec_Trade.py         # 队列消费（根据order_type分发）
└── applications\api\business\Queue_Business.py  # 入队（保留order_type和price字段）
```

### 同花顺操作差异

| 操作 | 市价委托 | 限价委托 |
|------|---------|---------|
| 菜单路径 | `市价委托 → 买入/卖出` | `买入[F1]` / `卖出[F2]` |
| 重填按钮 | ✅ 点击重填按钮（0x3EF） | ❌ 不存在，需手动清空输入框 |
| 输入股票代码 | 重填后直接输入 | 先清空代码框再输入 |
| 输入价格 | ❌ 不需要 | ✅ 先清空自动填入的市价再输入委托价 |
| 申报方式ComboBox | ✅ 选择五档等 | ❌ 不存在，直接点买入/卖出 |
| 点击按钮 | 买入/卖出（0x3EE） | 买入/卖出（0x3EE） |
| 确认弹窗 | "您是否确定以上市价买入/卖出委托" | "您是否确定以上限价买入/卖出委托" |
| 策略编号 | `agent` | `agent_limit` |

### 队列分发逻辑

`Timer_Exec_Trade.py` 根据 `order_type` 字段分发：
- `order_type == 'limit'` → `Exec_Limit_Trade.exec_run(item)` → `THS_LimitTraderServer`
- `order_type == 'market'`（默认）→ `Exec_Auto_Trade.exec_run(item)` → `THSTraderServer`

### 限价委托服务端操作流程

`THS_Limit_Trader_Server._trade_impl()` 完整流程：

1. 切换到限价委托界面（`买入[F1]` / `卖出[F2]`）
2. 清空并输入证券代码（0x408）
3. 处理市场二义性弹窗
4. 清空并输入委托价格（0x409）
5. 清空并输入委托数量（0x40A）
6. 校验可买/可卖数量
7. 点击买入/卖出按钮（0x3EE）
8. 处理委托确认弹窗（自动按Y）
9. 读取结果文本
10. 关闭结果弹窗
11. ESC退出

### 限价委托输入框清空策略

限价界面没有重填按钮，采用三重策略确保清空：
1. `set_edit_text('')` — 最可靠，直接设置
2. `^a{BACKSPACE}` — Ctrl+A全选 + Backspace删除
3. 逐字符 `{BACKSPACE}` — 按字符数逐个删除

## 科创板委托规则

- **仅支持限价委托**（市价委托需保护限价，脚本已禁用科创板市价委托）
- 最小申报数量200股，超过200股可按1股递增
- 限价委托时必须提供委托价格
- 涨跌幅±20%
- 连续竞价价格笼子：买入≤基准价102%，卖出≥基准价98%

## 北交所委托规则

- **仅支持限价委托**，不支持市价委托
- 必须提供委托价格，否则报错
- 涨跌幅±30%
- 连续竞价价格笼子：买入≤基准价105%或+0.1元，卖出≥基准价95%或-0.1元

## 常见错误示例

- ❌ `buy("688262", "国芯科技", 100)` → 科创板最小200股
- ❌ `buy("688262", "国芯科技", 200)` → 科创板不支持市价委托
- ❌ `limit_buy("688262", "国芯科技", 200)` → 缺少price参数
- ❌ `buy("830xxx", "北交所股", 100)` → 北交所不支持市价委托
- ❌ `limit_buy("830xxx", "北交所股", 100, price=None)` → price不能为None
- ✅ `limit_buy("688262", "国芯科技", 200, 35.78)` → 科创板限价委托
- ✅ `limit_buy("688262", "国芯科技", 201, 35.78)` → 科创板超200股可按1股递增
- ✅ `limit_buy("830001", "北交所股", 100, 10.00)` → 北交所限价委托
- ✅ `buy("300666", "江丰电子", 100)` → 创业板市价委托
- ✅ `limit_buy("300666", "江丰电子", 100, 25.50)` → 创业板限价委托
